## 一、项目技术栈分析

| 层级 | 技术 |
|------|------|
| 语言 | Kotlin 1.9 |
| UI | Jetpack Compose + Material3 |
| 架构 | Clean Architecture + MVVM |
| 数据库 | Room |
| DI | Hilt |
| 异步 | Kotlin Coroutines + Flow |
| AI 服务 | DeepSeek / Minimax API（OCR + 答案分析） |
| 备份 | 百度网盘 / 夸克网盘 |
| 平台 | Android（小米6 Pro平板） |

## 二、核心推荐

### 2.1 必装 Superpowers Skills

这些技能覆盖了项目的完整开发流程：

| Skill | 用途 | 触发场景 |
|-------|------|----------|
| `superpowers:brainstorming` | 需求分析与设计 | 开发任何功能前 |
| `superpowers:writing-plans` | 撰写实现计划 | 需求确认后 |
| `superpowers:subagent-driven-development` | 子任务并行执行 | 多模块同时开发 |
| `superpowers:test-driven-development` | 测试驱动开发 | 编写业务逻辑代码 |
| `superpowers:systematic-debugging` | 系统化调试 | 遇到 bug 时 |
| `superpowers:verification-before-completion` | 完成前验证 | 提交代码前 |
| `superpowers:finishing-a-development-branch` | 分支完成处理 | PR 合并前 |
| `superpowers:requesting-code-review` | 代码评审 | 代码审查 |

### 2.2 项目管理

| Skill | 用途 |
|-------|------|
| `superpowers:using-git-worktrees` | 多任务隔离（同时开发不同模块） |

### 2.3 文档与代码质量

| Skill | 用途 |
|-------|------|
| `superpowers:writing-skills` | 编写自定义技能 |
| `superpowers:receiving-code-review` | 处理评审反馈 |

## 四、快速开发流程建议

```
1. 需求分析 → superpowers:brainstorming
2. 制定计划 → superpowers:writing-plans
3. 项目骨架 → superpowers:subagent-driven-development（并行创建多个基础文件）
4. 数据层开发 → superpowers:test-driven-development（Room + Repository）
5. 领域层开发 → 同上
6. UI 层开发 → Compose 组件
7. AI 服务接入 → 调试模式
8. 集成测试 → superpowers:verification-before-completion
9. 代码评审 → superpowers:requesting-code-review
10. 分支完成 → superpowers:finishing-a-development-branch
```

## 五、环境配置建议

### 5.1 settings.json 配置

```json
{
  "permissions": {
    "allow": [
      "bash:./gradlew assembleDebug",
      "bash:./gradlew test",
      "bash:adb devices",
      "read:C:/Users/HP/Desktop/ppbj/**",
      "write:C:/Users/HP/Desktop/ppbj/**"
    ]
  },
  "hooks": {
    "before-command": "./gradlew assembleDebug"
  }
}
```

### 5.2 环境变量

```
# AI 服务
DEEPSEEK_API_KEY=xxx
MINIMAX_API_KEY=xxx

# 备份
BAIDU_APP_ID=xxx
BAIDU_APP_SECRET=xxx
```
